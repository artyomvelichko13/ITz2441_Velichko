# 🔇 Как отключить предупреждения OpenCV

## 🚀 БЫСТРОЕ РЕШЕНИЕ (для текущего запуска)

Если скрипт **УЖЕ работает** и вы хотите просто скрыть предупреждения:

### Способ 1: Переменная окружения (Windows)

```powershell
$env:OPENCV_LOG_LEVEL="ERROR"
python add_masks_to_spacenet.py
```

Или в одну строку:
```powershell
$env:OPENCV_LOG_LEVEL="ERROR"; python add_masks_to_spacenet.py
```

---

## 🔧 ПОСТОЯННОЕ РЕШЕНИЕ

### Способ 2: Автоматическое исправление всех скриптов

```powershell
python disable_opencv_warnings.py
```

Этот скрипт автоматически добавит подавление предупреждений во все файлы проекта.

---

### Способ 3: Ручное исправление

Откройте файл `add_masks_to_spacenet.py` и добавьте **В САМОЕ НАЧАЛО** (после docstring):

```python
"""
Создание масок для СУЩЕСТВУЮЩИХ изображений SpaceNet
Не удаляет скачанные изображения, только добавляет маски к ним!
"""

# ДОБАВЬТЕ ЭТИ СТРОКИ ⬇️
import os
import warnings

os.environ['OPENCV_LOG_LEVEL'] = 'ERROR'
warnings.filterwarnings('ignore')
# ⬆️ ДО ЭТОЙ СТРОКИ

import numpy as np
import cv2
from pathlib import Path
from tqdm import tqdm
```

Сохраните файл и запустите снова.

## ⚡ Для СЕЙЧАС (скрипт работает):

Откройте **НОВОЕ окно PowerShell** и выполните:

```powershell
# Установите переменную окружения
$env:OPENCV_LOG_LEVEL="ERROR"

# Проверьте (должно вывести "ERROR")
echo $env:OPENCV_LOG_LEVEL

# Теперь запустите скрипт
python add_masks_to_spacenet.py
```

**Предупреждения исчезнут!** ✨

## ❓ FAQ

**Q: Влияют ли эти предупреждения на работу?**  
A: Нет, это просто информационные сообщения. Скрипт работает корректно.

**Q: Почему они появляются?**  
A: SpaceNet использует GeoTIFF формат с географическими метаданными. OpenCV их не понимает, но предупреждает.

**Q: Безопасно ли отключать предупреждения?**  
A: Да, мы отключаем только TIFF предупреждения, не критичные ошибки.

**Q: Нужно ли перезапускать скрипт?**  
A: Если используете переменную окружения - нет. Если правите файл - да.
