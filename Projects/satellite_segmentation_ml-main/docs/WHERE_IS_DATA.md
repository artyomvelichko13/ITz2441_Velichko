# 📁 ГДЕ НАХОДЯТСЯ ДАННЫЕ?

## Текущая ситуация

Когда вы запускаете `demo_pipeline.py` или `train_segmentation.py`, скрипт:

1. **Проверяет наличие папки `data/`** в текущей директории
2. **Если данных нет** → автоматически создаёт синтетические данные **в памяти** (не сохраняет на диск)
3. **Если данные есть** → использует их для обучения

## 🎯 Как создать локальные данные (на диске)

### Способ 1: Автоматическая генерация (Рекомендуется)

```bash
python create_local_data.py
```

Это создаст:
```
C:\Users\Dmitry\Desktop\neuroLabs\lab1\
├── data\                          ← НОВАЯ ПАПКА
│   ├── images\                    ← 500 изображений
│   │   ├── synthetic_0000.png
│   │   ├── synthetic_0001.png
│   │   └── ...
│   ├── masks\                     ← 500 масок
│   │   ├── synthetic_0000.png
│   │   ├── synthetic_0001.png
│   │   └── ...
│   ├── README.txt
│   └── example_sample.png         ← Пример данных
```

### Способ 2: Использование prepare_data.py

```bash
python prepare_data.py
```

Выберите опцию **2** для создания синтетических данных.

### Способ 3: Ручное создание папок

```bash
mkdir data
mkdir data\images
mkdir data\masks
```

Затем положите туда свои изображения и маски.

## 🔍 Проверка наличия данных

После создания данных проверьте:

```bash
# Windows PowerShell

dir data\masks

# Должно быть по 500 файлов в каждой папке
```

## 🚀 Запуск обучения с локальными данными

После создания папки `data/`:

```bash

python demo_pipeline.py

# Полное обучение (20 эпох)
python train_segmentation.py
```

Теперь скрипт обнаружит папку `data/` и использует локальные файлы!

## 💡 Разница между режимами

### Режим 1: БЕЗ папки data/ (текущий)
```python
# Данные создаются в памяти, НЕ сохраняются на диск
train_dataset = SyntheticDataset(num_samples=200)
```
- ✅ Быстрый старт
- ❌ Данные генерируются каждый раз заново
- ❌ Нельзя посмотреть данные

### Режим 2: С папкой data/ (рекомендуется)
```python
# Данные читаются с диска
train_dataset = SpaceNetDataset(
    image_dir='data/images',
    mask_dir='data/masks'
)
```
- ✅ Данные сохранены на диске
- ✅ Можно посмотреть и проверить
- ✅ Быстрее при повторном запуске

## ❓ FAQ

**Q: Почему я не вижу папку data/?**  
A: Она не создаётся автоматически. Запустите `python create_local_data.py`

**Q: Нужно ли каждый раз создавать данные заново?**  
A: Нет! После создания папки `data/`, данные сохраняются и используются повторно.

**Q: Сколько места занимают данные?**  
A: Примерно 500 МБ для 500 примеров размером 512×512.

**Q: Могу ли я добавить свои изображения?**  
A: Да! Положите их в `data/images/` и `data/masks/` в формате PNG.

## 🔧 Устранение проблем

**Проблема:** Скрипт не видит данные  
**Решение:** Проверьте структуру:
```
data\
├── images\
│   └── *.png  (изображения)
└── masks\
    └── *.png  (маски с теми же именами)
```

**Проблема:** Ошибка при создании данных  
**Решение:** Проверьте права на запись в папку и наличие свободного места.
