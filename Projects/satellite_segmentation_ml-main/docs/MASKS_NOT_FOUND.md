## Почему так произошло?

**SpaceNet SN8 Floods использует ВЕКТОРНЫЕ аннотации (GeoJSON), а не готовые растровые маски!**

### Что в датасете:
- ✅ Спутниковые снимки (.png / .tif)
- ✅ Векторные аннотации зданий (.geojson)
- ✅ Векторные аннотации наводнений (.geojson)
- ❌ Растровые маски для сегментации (НЕТ!)

### Что нужно для обучения:
- ✅ Изображения
- ❌ **Растровые маски** (пиксельная разметка классов)

## 🔧 Техническое объяснение

### Почему программа не нашла маски?

В `download_spacenet_windows.py` (строки 70-90):

```python
# Ищем файлы
if 'mask' in file.lower() or 'label' in file.lower():
    mask_files.append(file_path)
```

**Проблема:** SpaceNet не содержит файлов с названиями `*mask*` или `*label*` в формате .png/.tif

**Что есть на самом деле:**
- `building-labels.geojson` - векторные полигоны зданий
- `flood-labels.geojson` - векторные полигоны наводнений
- `pre-event.tif` - изображение до события
- `post-event.tif` - изображение после события

## ❓ FAQ

**Q: Можно ли использовать SpaceNet для сегментации?**  
A: Да, но нужно конвертировать GeoJSON → растры. Это требует дополнительной работы.

**Q: Синтетические данные подходят для обучения?**  
A: Да! Они специально созданы для демонстрации работы модели U-Net.

**Q: Какой датасет лучше?**  
A: Для быстрого старта - синтетические. Для реального проекта - DeepGlobe или Inria.

**Q: Удалить скачанные 7 ГБ SpaceNet?**  
A: Можете удалить папки `data_raw` и `data_extracted` если не планируете конвертировать GeoJSON.
