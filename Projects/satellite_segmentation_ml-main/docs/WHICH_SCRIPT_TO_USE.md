# ⚠️ ВАЖНО: Разница между скриптами

## 🔴 Вопрос: Что произойдёт со скачанными изображениями SpaceNet?

## Ответ зависит от того, какой скрипт вы запустите:

---

## 📊 Сравнение скриптов

| Скрипт | Удалит SpaceNet? | Что создаст? | Результат |
|--------|-----------------|--------------|-----------|
| `create_local_data.py` | ❌ ДА | Новые синтетические изображения + маски | SpaceNet ПЕРЕЗАПИСАНЫ |
| `add_masks_to_spacenet.py` | ✅ НЕТ | Только маски для ваших изображений | SpaceNet СОХРАНЕНЫ + маски |

---

## Вариант 1: `create_local_data.py` ❌

### Что делает:
```powershell

```

### ДО:
```

├── images/  (4346 файлов SpaceNet)  ✅
└── masks/   (0 файлов)               ❌
```

### ПОСЛЕ:
```
data/
├── images/  (500 синтетических)     ⚠️ SpaceNet удалены!
└── masks/   (500 синтетических)     ✅
```

### Когда использовать:
- ❌ **НЕ используйте**, если хотите сохранить SpaceNet!
- ✅ Используйте, если хотите начать заново с синтетическими данными

## ❓ FAQ

**Q: Маски будут настоящими?**  
A: Нет, маски синтетические, но созданы специально для ваших изображений SpaceNet. Это лучше, чем ничего!

**Q: Можно ли использовать create_local_data.py?**  
A: Да, но вы ПОТЕРЯЕТЕ скачанные 4346 изображений SpaceNet (7 ГБ).

**Q: Что если я уже запустил create_local_data.py?**  
A: Скачайте SpaceNet заново: `python download_spacenet_windows.py`

**Q: Почему не использовать настоящие маски SpaceNet?**  
A: SpaceNet использует векторные аннотации (GeoJSON), а не растровые маски. Конвертация сложная.

**Q: Удалить скачанные 7 ГБ после создания масок?**  
A: Можете удалить папки `data_raw` и `data_extracted`, но НЕ удаляйте `data/`!

## 🗑️ Что можно удалить для экономии места

После запуска `add_masks_to_spacenet.py`:

```powershell
# Удалить архивы (освободит ~7 ГБ)
rmdir /s /q data_raw

# Удалить распакованные файлы (освободит ~7 ГБ)
rmdir /s /q data_extracted
```

**НЕ удаляйте:**
- `data/` - здесь ваши рабочие данные!
- `data/images/` - ваши изображения
- `data/masks/` - созданные маски
